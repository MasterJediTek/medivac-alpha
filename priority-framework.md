# Priority Framework for Outstanding Items

## Critical Path (v1.0 Release)

**Timeline:** 1–2 weeks  
**Effort:** 10–15 days  
**Impact:** Blocks release if not completed

Items in this category are bugs, missing critical features, or code quality issues that prevent the project from being released as v1.0. Examples include:

- Known bugs affecting core workflows (e.g., text input fields not accepting input)
- UI/UX issues affecting usability (e.g., icons not displaying, buttons not responsive)
- Build errors preventing deployment (e.g., TypeScript errors, missing dependencies)
- Code quality issues that reduce maintainability (e.g., unused code, code duplication)

**Action:** Must complete all items in this category before release.

## High Priority (v1.0+ Roadmap)

**Timeline:** 3–4 weeks  
**Effort:** 15–20 days  
**Impact:** Recommended for early adoption; can ship in v1.0.1 if necessary

Items in this category are high-value features that complete core systems or significantly improve user experience. Examples include:

- Features that complete partially-implemented systems (e.g., route sharing notifications completing the route sharing feature)
- Features that improve reliability (e.g., offline caching for better connectivity)
- Features that enhance security or transparency (e.g., activity logging for auditing)

**Action:** Should implement before v1.0 if time permits; otherwise schedule for v1.0.1.

## Medium Priority (v1.1+ Roadmap)

**Timeline:** 6–8 weeks  
**Effort:** 40–50 days  
**Impact:** Valuable features; recommended for v1.1 release

Items in this category are features that enhance existing systems or add new capabilities. Examples include:

- Advanced features that build on core systems (e.g., beacon calibration, route history panel)
- Features that improve accessibility (e.g., accessibility audio descriptions)
- Features that improve administration (e.g., analytics dashboards)

**Action:** Schedule for v1.1 release after v1.0 is stable.

## Lower Priority (v1.2+ Roadmap)

**Timeline:** 12–16 weeks  
**Effort:** 80–100 days  
**Impact:** Nice-to-have features; can be deferred to future releases

Items in this category are advanced features that are not essential for core functionality. Examples include:

- Gamification features (e.g., cooperative mission system)
- Immersive experiences (e.g., VR, avatar navigation)
- Advanced media features (e.g., emergency broadcast system, picture-in-picture)

**Action:** Document in product roadmap; schedule for v1.2 or later based on business priorities.

## Categorisation Criteria

Use these criteria to categorise outstanding items:

| Criterion | Critical | High | Medium | Low |
|-----------|----------|------|--------|-----|
| **Blocks Release** | Yes | No | No | No |
| **Affects Core Workflow** | Yes | Yes | No | No |
| **Improves UX/Reliability** | Yes | Yes | Yes | No |
| **Effort (days)** | 1–3 | 3–5 | 4–6 | 5+ |
| **User Impact** | High | High | Medium | Low |
| **Business Value** | Critical | High | Medium | Low |
